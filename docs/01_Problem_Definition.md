# Проблема и границы проекта

## 1. Проблема

Отсутствие системы мониторинга приводит к тому, что:
- Перегрузки обнаруживаются только в момент аварии
- Нельзя отследить, какие приборы потребляют больше всего
- Невозможно доказать, кто именно устроил перегрузку
- Повышается риск пожаров из-за неисправных приборов

## 2. Влияние на пользователей

| Кто         | Как страдает                                                 |
|-------------|--------------------------------------------------------------|
| Охрана      | Не могут предотвратить аварию, только реагировать постфактум |
| Комендант   | Тратит время на поиск причин, растут счета                   |
| Студенты    | Плата за электричество распределяется вслепую                |
| Университет | Риск пожаров, лишние расходы                                 |

## 3. Целевое состояние

Система в реальном времени:
- Отслеживает потребление по комнатам/этажам
- Предупреждает охрану в Telegram при аномалиях
- Накапливает статистику для анализа
- Помогает находить неисправные приборы до пожара

## 4. Границы проекта

### ✅ Входит в scope (текущая версия 0.2.0)
- Мониторинг электропотребления в одном корпусе (пилот)
- Сбор данных (первая версия — симуляция)
- Детекция аномалий по порогам
- Уведомления в Telegram
- Экспорт истории в CSV

### ❌ НЕ входит в текущую версию (НО планируется)
- ☁️ **Облачное хранение** 
- 📈 **Визуализация в Grafana** 
- 🔌 **Реальные датчики ESP32** 
- 🤖 **Автоотключение**

📌 *Подробности — в [Roadmap](./06_Roadmap.md)*                                            

## 5. Пользователи системы

| Роль      | Потребности                    | Как используют       |
|-----------|--------------------------------|----------------------|
| Охранник  | Быстро узнавать об аномалиях   | Telegram-уведомления |
| Комендант | Анализировать статистику       | CSV-отчеты           |
| Энергетик | Выявлять неисправные приборы   | Графики потребления  |
| Студсовет | Понимать справедливость счетов | Общая статистика     |

## 6. Критерии успеха 

- [ ] Время обнаружения аномалии < 5 минут
- [ ] 90% перегрузок детектятся до отключения автоматов
- [ ] Охрана может пользоваться системой без инструкции (интуитивно)
- [ ] За год собрана статистика по всем комнатам
- [ ] Снижение среднего счета на 20-30% за счет выявления "прожорливых" приборов
