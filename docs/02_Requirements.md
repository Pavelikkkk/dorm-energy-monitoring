# Требования к системе

## 1. Функциональные требования (MoSCoW)

### 🔴 MUST HAVE 

- [ ] **Генерация данных** — создание синтетических данных за указанный период (—days)
- [ ] **Детекция по порогам** — обнаружение превышения заданного значения (—threshold)
- [ ] **Сохранение в CSV** — экспорт результатов в файл (—output)
- [ ] **CLI-интерфейс** — запуск с параметрами командной строки

### 🟡 SHOULD HAVE 

- [ ] **Учет времени суток** — разные паттерны для дня/ночи
- [ ] **Учет выходных** — пониженное потребление в выходные
- [ ] **Telegram-уведомления** — отправка оповещений при аномалиях
- [ ] **Детекция ночного падения** — аномально низкое потребление (2–5 утра)

### 🟢 COULD HAVE 

- [ ] **ML-детектор** — инференс ONNX-модели
- [ ] **Другие мессенджеры** (WhatsApp/Viber)
- [ ] **Визуализация в консоли** (псевдографика)
- [ ] **Экспорт в JSON**

## 3. Вывод данных (Outputs)

| Назначение | Формат           | Получатель  | Триггер              |
|------------|------------------|-------------|----------------------|
| Архив      | CSV              | Комендант   | По запросу (—output) |

| Логи       | Текст (файл)     | Разработчик | Всегда               |

## 4. Нефункциональные требования

### ⚡ Производительность

| Параметр                | Цель                    |
|-------------------------|-------------------------|
| Генерация 1 года данных | < 1 секунды             |
| Telegram-уведомление    | < 5 секунд от аномалии  |

### 🛡️ Надежность

| Параметр        | Требование                                   |
|-----------------|----------------------------------------------|
| Логирование     | Все ошибки в `error.log`                     |
| Обработка сбоев | При падении Telegram — локальное логирование |
| Доступность     | 99% (кроме интернета)                        |

### 💾 Ресурсы

| Ресурс | Оценка                     |
|--------|----------------------------|
| RAM    | < 100 MB                   |
| Диск   | 1 MB на месяц данных (CSV) |
| Сеть   | 1 KB на уведомление        |

## 5. Внешние интерфейсы

## 6. Ограничения

### Технические
- **Язык:** C++20 (стандарт)
- **Сборка:** CMake 3.15+
- **Платформы:** 
  - Linux (GCC 13+, Clang 16+) 
  - Windows (MSVC 2022, MinGW) 
  - macOS (Clang 16+) 

Код написан с использованием только стандартной библиотеки и не содержит платформенно-зависимых вызовов, поэтому **переносим между платформами без изменений**.
