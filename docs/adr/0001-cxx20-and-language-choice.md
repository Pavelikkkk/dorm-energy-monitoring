# ADR-001: Выбор языка программирования и стандарта C++20

## 1. Контекст и требования

Система предназначена для непрерывного мониторинга на сервере общежития с ограниченными ресурсами. Ключевые требования:
- **Время реакции:** ≤ 5 секунд (от датчика до оповещения).
- **Надежность:** Работа без перезагрузок и деградации производительности в течение длительных периодов.
- **Ресурсы:** Минимальное потребление CPU и RAM.

## 2. Принятое решение

Для реализации backend-части системы выбран язык **C++20**.

## 3. Обоснование выбора

Выбор C++20 продиктован необходимостью обеспечить **детерминированное время отклика** и **минимальный оверхед** на оборудовании с ограниченными мощностями.

### 3.1. Предсказуемость времени отклика
*   **Отсутствие Garbage Collector:** В отличие от Python или Java, C++ не имеет сборщика мусора, который может вызывать непредсказуемые паузы. Освобождение памяти происходит детерминированно благодаря механизму RAII.
*   **Контроль над concurrency:** Мы явно управляем потоками и приоритетами. Критические задачи (обработка MQTT, инференс) не блокируются фоновыми процессами.
*   **Async I/O:** Использование библиотеки `asio` позволяет обрабатывать тысячи подключений в event-loop без накладных расходов на переключение контекста ОС, обеспечивая стабильный P95 latency < 100 мс.

### 3.2. Эффективность использования ресурсов
*   **Низкий footprint:** Скомпилированный бинарный файл не требует тяжелого интерпретатора. Потребление RAM в idle-режиме ожидается на уровне ~50–100 МБ.

### 3.3. Надежность
*   **RAII и умные указатели:** Автоматическое управление временем жизни объектов исключает утечки памяти и дескрипторов.
*   **Статическая линковка:** Возможность поставки автономного бинарного файла упрощает деплой и устраняет «dependency hell».

### Почему отвергнуты другие варианты?
1.  **Python:** Отклонен из-за риска нарушения SLA (≤5 с) при пиковых нагрузках. GIL и недетерминированный GC делают невозможным гарантированное время отклика на слабом железе без сложной архитектуры мультипроцессности.
2.  **Rust:** Отклонен в пользу C++20 из-за более зрелой документации по интеграции специфичных промышленных протоколов и наличия готовых примеров для ONNX Runtime в C++.
